How to use the Knowledge Graph

Navigation

DragRotate the 3D graph
ScrollZoom in/out
Click nodeFocus node — highlights its direct neighbours, flies the camera to it, and opens the detail panel with note content
✦ buttonOpen the AI Copilot panel to chat about the graph

Visualization controls

Color byCommunity — each Leiden community gets a unique colour. Type — coloured by node type (concept, person, source…). Centrality — gradient from low to high betweenness (bridge importance).
Size byDegree — larger nodes have more connections. Betweenness — larger nodes are more often on shortest paths between others.
Edge styleWeight — edge thickness reflects relationship strength. Bridges — cross-community edges glow orange; intra-community edges fade. Uniform — flat equal-width edges.

Filters & overlays

Node typesToggle concept / person / source / project / synthesis / entity visibility.
Hub nodesHighlights the 10 nodes with highest betweenness centrality — the intellectual bridges of the vault.
Isolated nodesHighlights nodes not connected to the main component — orphaned ideas to reconnect.
Link hintsShows predicted missing links (Adamic-Adar algorithm) as amber dashed-style edges — synthesis candidates.
CommunitiesClick a community chip to isolate and inspect only that cluster. Click again to deselect.

Pathfinding

From / ToType note titles (autocomplete available). Hit Find path to show only the nodes on the shortest Dijkstra path. Hit Reset path to restore the full graph.

AI Copilot

✦ CopilotChat with the graph. The Copilot knows your current view context — selected node, active path, filters. Use Local (Ollama) or Cloud (OpenRouter) models. Click suggestion chips for follow-up questions.
